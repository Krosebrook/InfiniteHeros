# Contributing Guidelines

We welcome contributions to Infinite Heroes! Please follow these guidelines to ensure a smooth collaboration process.

## Getting Started
1. Fork the repository.
2. Clone your fork locally.
3. Install dependencies: `npm install`
4. Start the development server: `npm run dev`

## Code Style
- **Framework:** React 19 (Functional components and Hooks).
- **Styling:** Tailwind CSS. Avoid custom CSS unless absolutely necessary.
- **TypeScript:** Use strict typing. Define interfaces in `types.ts`.
- **Linting:** Run `npm run lint` before submitting a PR to ensure there are no TypeScript errors.

## Pull Request Process
1. Create a new branch for your feature or bugfix: `git checkout -b feature/my-new-feature`
2. Make your changes and commit them with clear, descriptive messages.
3. Push your branch to your fork: `git push origin feature/my-new-feature`
4. Open a Pull Request against the `main` branch of the original repository.
5. Provide a detailed description of your changes in the PR template.

## Reporting Bugs
If you find a bug, please open an issue with the following information:
- A clear description of the problem.
- Steps to reproduce the bug.
- Expected behavior vs. actual behavior.
- Browser and OS version.
- Console errors (if any).
