# Security Policy

## API Key Handling
Infinite Heroes is designed as a client-side application. It requires users to provide their own Google Gemini API key.

- **Storage:** The API key is stored in the browser's `localStorage` for convenience across sessions. It is **never** transmitted to any server other than Google's official API endpoints.
- **Risk:** Because the key is stored in the browser, it is accessible to any scripts running on the same domain. Users should only use this application on trusted networks and devices.
- **Recommendation:** Users are strongly advised to create a dedicated API key for this application and set appropriate quota limits in their Google Cloud Console to prevent unexpected charges in case the key is compromised.

## Content Safety
The application relies on the safety filters built into the Google Gemini API to prevent the generation of harmful, explicit, or policy-violating content.

- **Text Generation:** The AI is instructed to adhere to the selected story tone, but users can input custom premises. The Gemini API will block responses that violate its safety guidelines.
- **Image Generation:** The `gemini-3.1-flash-image-preview` model includes robust safety checks. If an image prompt (even one generated by the AI itself) triggers these filters, the image will not be generated, and the app will handle the error gracefully.

## Reporting Vulnerabilities
If you discover a security vulnerability in the application code, please do not open a public issue. Instead, contact the maintainers directly via email (KyleRosebrook@gmail.com) with a detailed description of the issue and steps to reproduce it.
